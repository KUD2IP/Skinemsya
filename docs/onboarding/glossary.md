# Onboarding: Glossary

## Purpose

Краткий словарь для быстрого старта. Полный словарь — в `business/glossary.md`.

## Context

Новый разработчик должен быстро понять ключевые термины без чтения полного glossary.

## Responsibilities

- Дать сокращенный список терминов MVP.
- Сослаться на полный glossary.

## Non Responsibilities

- Документ не заменяет `business/glossary.md`.

## Design Decisions

| Термин | Краткое значение |
| --- | --- |
| Группа | Контейнер для мероприятий. Типы: CHAT_LINKED (из чата) или STANDALONE (вручную). |
| Мероприятие (Event) | Конкретный совместный расход внутри группы. |
| Плательщик | Участник, оплативший чек вне системы. |
| Позиция | Строка расхода: название, количество, цена. |
| Долг (Debt) | Сумма, которую участник должен плательщику. |
| Платеж (Payment) | Процесс закрытия долга: перевести → перевел → получил. |
| Чек (Receipt) | Изображение чека → ML → позиции. Опционально. |
| Реквизиты | Данные для перевода, хранятся в профиле пользователя. |
| Владелец (Owner) | Создатель группы, может редактировать и удалять. |
| MVP | Минимальная версия без микросервисов, Kafka, СБП. |

Полный словарь: `docs/business/glossary.md`.

## Constraints

- Только MVP-термины.
- Без технических деталей (таблицы, классы).
