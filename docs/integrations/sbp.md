# Integration: SBP

## Purpose

Документ описывает будущую интеграцию с Системой быстрых платежей (СБП) и условия перехода от ручных переводов.

## Context

MVP использует ручной перевод по реквизитам с двухсторонним подтверждением (ADR-0007). СБП рассматривается как следующий этап для автоматизации платежей.

## Responsibilities

- Описать ожидаемый контракт СБП-интеграции.
- Зафиксировать входные и выходные данные.
- Определить статусы СБП-платежа.
- Описать условия перехода от ручного сценария.

## Non Responsibilities

- Документ не реализует СБП на MVP.
- Документ не выбирает конкретного банка-партнера.
- Документ не описывает юридические требования.

## Design Decisions

Условия перехода к СБП:
1. MVP с ручными платежами стабилен и используется пользователями.
2. Выбран банк-партнер с API СБП.
3. Определены требования к безопасности и хранению данных.

Ожидаемый поток (post-MVP):
1. Должник инициирует оплату через СБП в Mini App.
2. Backend создает СБП-платеж через `SbpAdapter`.
3. Пользователь подтверждает в банковском приложении.
4. Backend получает callback/webhook о статусе.
5. Долг закрывается автоматически при успешном платеже.

Статусы: `INITIATED`, `PENDING`, `COMPLETED`, `FAILED`, `EXPIRED`.

## Constraints

- СБП не является MVP-зависимостью.
- Ручной сценарий остается fallback.
- Секреты СБП — отдельные env vars.
- PCI/банковские требования оцениваются при выборе партнера.

## Future Evolution

- Полная замена ручного сценария.
- QR-код для оплаты.
- Автоматическая сверка без подтверждения плательщика.
